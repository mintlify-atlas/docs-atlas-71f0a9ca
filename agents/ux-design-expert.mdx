**Design tokens example**:

```yaml
colors:
  primary:
    50: "#f0f9ff"
    500: "#0ea5e9"
    900: "#0c4a6e"
  
  semantic:
    success: "{colors.primary.500}"
    error: "#ef4444"
    warning: "#f59e0b"

spacing:
  xs: "0.25rem"   # 4px
  sm: "0.5rem"    # 8px
  md: "1rem"      # 16px
  lg: "1.5rem"    # 24px
  xl: "2rem"      # 32px

typography:
  font_family:
    sans: "Inter, system-ui, sans-serif"
    mono: "'Fira Code', monospace"
  
  font_size:
    xs: "0.75rem"   # 12px
    sm: "0.875rem"  # 14px
    base: "1rem"    # 16px
    lg: "1.125rem"  # 18px
    xl: "1.25rem"   # 20px
```

### Phase 4: Atomic Component Building

```bash
# Build atom (base component)
*build button

# Uma will:
# - Create component file (TypeScript + React)
# - Define props interface
# - Implement variants (primary, secondary, ghost)
# - Add accessibility (ARIA labels, keyboard nav)
# - Write unit tests
# - Generate Storybook story
# - Document usage

# Compose molecule (combination of atoms)
*compose form-field

# Uma will:
# - Combine atoms (Label + Input + ErrorMessage)
# - Define molecule props
# - Handle internal state
# - Add validation logic
# - Write integration tests
# - Document composition pattern
```

**Atomic Design hierarchy**:

<Tabs>
  <Tab title="Atoms">
    **Base components** (cannot be broken down further)
    
    Examples:
    - Button
    - Input
    - Label
    - Icon
    - Badge
    
    **Characteristics**:
    - Single responsibility
    - Highly reusable
    - Minimal props
    - No business logic
  </Tab>
  
  <Tab title="Molecules">
    **Simple combinations** of atoms
    
    Examples:
    - FormField (Label + Input + Error)
    - SearchBox (Input + Icon + Button)
    - Card (Image + Title + Description)
    
    **Characteristics**:
    - Combines 2-4 atoms
    - Reusable patterns
    - Basic composition logic
    - Minimal state
  </Tab>
  
  <Tab title="Organisms">
    **Complex UI sections** with business logic
    
    Examples:
    - Header (Logo + Navigation + SearchBox)
    - DataTable (filters + pagination + rows)
    - UserProfile (Avatar + Info + Actions)
    
    **Characteristics**:
    - Combines molecules and atoms
    - Business logic included
    - More specific to use case
    - Stateful components
  </Tab>
  
  <Tab title="Templates & Pages">
    **Layout structures** and **specific instances**
    
    **Templates**: Page layouts without real content
    **Pages**: Templates with actual data
    
    Examples:
    - DashboardTemplate → DashboardPage
    - ProfileTemplate → UserProfilePage
  </Tab>
</Tabs>

**Accessibility checklist (WCAG AA)**:

- ✅ Color contrast ≥ 4.5:1 for text
- ✅ Keyboard navigation works for all interactions
- ✅ Focus indicators visible
- ✅ ARIA labels on interactive elements
- ✅ Semantic HTML (headings, landmarks)
- ✅ Alt text on images
- ✅ Form labels associated with inputs
- ✅ Error messages announced to screen readers

## W3C Design Tokens (DTCG)

```bash

# Uma will:
# - Convert tokens.yaml to DTCG JSON format
# - Generate token bundles
# - Create CSS custom properties
# - Generate Tailwind config
# - Support token references and aliases
```

**DTCG format example**:

```json
{
  "colors": {
    "primary": {
      "$type": "color",
      "$value": "#0ea5e9"
    },
    "semantic": {
      "success": {
        "$type": "color",
        "$value": "{colors.primary}"
      }
    }
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Audit finding too many patterns" icon="magnifying-glass">
    **Issue**: Codebase has extreme redundancy
    
    **Solutions**:
    1. Start with most impactful patterns (buttons, colors)
    2. Use consolidation algorithm to group similar patterns
    3. Generate shock report to visualize chaos
    4. Create phased migration plan
  </Accordion>
  
  <Accordion title="Accessibility score low" icon="universal-access">
    **Issue**: WCAG compliance below target
    
    **Solutions**:
    ```bash
    *a11y-check  # Get detailed report
    
    # Common fixes:
    # - Improve color contrast (≥4.5:1)
    # - Add ARIA labels to interactive elements
    # - Add keyboard navigation support
    # - Add alt text to images
    # - Use semantic HTML
    ```
  </Accordion>
  
  <Accordion title="Component building slow" icon="clock">
    **Issue**: Building components taking too long
    
    **Solutions**:
    1. Use Shadcn for base components (`*bootstrap-shadcn`)
    2. Start with atoms, compose molecules later
    3. Use design tokens to avoid hardcoding
    4. Leverage Storybook for isolated development
  </Accordion>
</AccordionGroup>
