# 🌊 River - Scrum Master

**Role**: Technical Scrum Master - Story Preparation Specialist  
**Archetype**: Facilitator ♓ Pisces  
**Persona**: Task-oriented, efficient, precise, focused on clear developer handoffs

## When to Use River

Activate `@sm` for:

- 📖 User story creation from PRD
- ✅ Story validation and completeness
- 🎯 Acceptance criteria definition
- 🔄 Story refinement
- 🏽 Sprint planning facilitation
- 🌱 Local branch management

**NOT for**: PRD creation (use `@pm`), Story implementation (use `@dev`), Remote git operations (use `@devops`)

## Story Creation Workflow

### Draft Next Story

```bash
*draft

# River will:
# 1. Read PRD and Architecture docs
# 2. Analyze next story in epic sequence
# 3. Create detailed story file
# 4. Define acceptance criteria (Given-When-Then)
# 5. Break down into tasks and subtasks
# 6. Embed CodeRabbit quality gates
# 7. Predict specialized agents
# 8. Run story draft checklist
```

### Story Structure

<Tabs>
  <Tab title="Story Header">

  <Tab title="Acceptance Criteria">
    ```gherkin
    ## Acceptance Criteria
    
    ### AC1: View Profile
    Given: User is logged in
    When: User navigates to /profile
    Then: Profile information is displayed
      - Full name
      - Email address
      - Avatar image
      - Join date
    
    ### AC2: Edit Profile
    Given: User is on profile page
    When: User clicks "Edit Profile" button
    Then: Form fields become editable
    And: "Save" and "Cancel" buttons appear
    
    ### AC3: Save Changes
    Given: User has edited profile fields
    When: User clicks "Save"
    Then: Changes are persisted to database
    And: Success message is displayed
    And: Form returns to read-only mode
    ```
  </Tab>

## Branch Management

### Local Branch Operations

<Info>
River manages **LOCAL** branches during development.  
@devops manages **REMOTE** operations (push, PR).
</Info>

**Allowed**:
```bash
# Create feature branch
git checkout -b feature/5.2-user-profile

# List branches
git branch

# Switch branches
git checkout main

# Delete local branch
git branch -d feature/5.2-user-profile

# Merge locally
git merge feature/5.2-user-profile
```

**Blocked** (use @devops):
```bash
git push                      # ❌ Use @devops
gh pr create                  # ❌ Use @devops
git push origin --delete      # ❌ Use @devops
```

### Development Workflow

```bash
# 1. River creates story
@sm *draft

# 2. River creates feature branch
git checkout -b feature/5.2-user-profile

# 3. Developer implements
@dev *develop story-5.2

# 4. River coordinates push
@devops *push
```

## Predictive Quality Planning

### Agent Prediction

River predicts which agents will be needed:

```yaml
story_5.2_user_profile:
  complexity: MEDIUM
  
  predicted_agents:
    dev:
      reason: "Primary implementation"
      tasks: ["API endpoints", "Frontend component"]
    
    ux-design-expert:
      reason: "UI design and components"
      tasks: ["Wireframes", "Design system components"]
    
    data-engineer:
      reason: "Profile schema may need updates"
      tasks: ["Schema review", "Migration if needed"]
    
    qa:
      reason: "Quality review and testing"
      tasks: ["Test strategy", "Accessibility check"]
```

### Quality Gate Assignment

River assigns appropriate gates based on story type:

```yaml
story_type: UI Feature

quality_gates:
  pre_commit:
    - CodeRabbit security scan
    - Linting
    - Unit tests
  
  pre_review:
    - CodeRabbit full review
    - Integration tests
    - Accessibility check (WCAG AA)
    - Performance budget (< 2s load time)
  
  pre_merge:
    - QA approval
    - All acceptance criteria met
    - Documentation updated
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Story too vague" icon="question">
    **Problem**: Developer doesn't know where to start
    
    **Solution**:
    - Add more technical notes
    - Break tasks into smaller subtasks
    - Reference specific files/components
    - Include code examples or patterns
  </Accordion>
  
  <Accordion title="Story too large" icon="arrows-maximize">
    **Problem**: Story estimated > 13 points
    
    **Solution**:
    - Split into 2+ stories
    - Identify MVP vs enhancements
    - Create separate stories for each user flow
  </Accordion>
  
  <Accordion title="Missing dependencies" icon="link-slash">
    **Problem**: Story blocked by other work
    
    **Solution**:
    - Document dependency in story
    - Update epic sequence
    - Coordinate with @po on prioritization
  </Accordion>
</AccordionGroup>
