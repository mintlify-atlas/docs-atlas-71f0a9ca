## Brainstorming Workflow

### Structured Ideation

```bash
*brainstorm "new feature ideas for user dashboard"

# Atlas will:
# 1. Choose brainstorming technique (e.g., SCAMPER, Six Thinking Hats)
# 2. Facilitate structured session
# 3. Capture all ideas without judgment
# 4. Help prioritize and cluster concepts
# 5. Generate brainstorming-output-tmpl.yaml
```

**Brainstorming techniques used**:

<Tabs>
  <Tab title="SCAMPER">
    **Framework**: Substitute, Combine, Adapt, Modify, Put to other uses, Eliminate, Reverse
    
    **Best for**: Product innovation, feature enhancement
    
    **Example questions**:
    - What can we substitute in the current flow?
    - How can we combine existing features?
    - What can we eliminate to simplify?
  </Tab>
  
  <Tab title="Six Thinking Hats">
    **Framework**: White (facts), Red (emotions), Black (caution), Yellow (optimism), Green (creativity), Blue (process)
    
    **Best for**: Comprehensive exploration, team alignment
    
    **Ensures**: All perspectives considered
  </Tab>
  
  <Tab title="Mind Mapping">
    **Framework**: Central idea with branching concepts
    
    **Best for**: Exploring relationships, visual thinkers
    
    **Output**: Hierarchical idea structure
  </Tab>
  
  <Tab title="5 Whys">
    **Framework**: Ask "why" 5 times to find root cause
    
    **Best for**: Problem analysis, uncovering needs
    
    **Output**: Deep understanding of user pain points
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Research taking too long" icon="clock">
    **Issue**: Exa search or research tasks timing out
    
    **Solutions**:
    1. Break research into smaller topics
    2. Use more specific search queries
    3. Set time limits per research area
  </Accordion>
  
  <Accordion title="Brainstorming not productive" icon="lightbulb">
    **Issue**: Session not generating useful ideas
    
    **Solutions**:
    1. Try different brainstorming technique
    2. Be more specific with topic framing
    3. Use divergent thinking first, convergent later
    4. Ensure psychological safety (no idea judging)
  </Accordion>
  
  <Accordion title="Research sources not credible" icon="shield-check">
    **Issue**: Concerns about data reliability
    
    **Solutions**:
    1. Verify sources are authoritative (gov, edu, industry leaders)
    2. Cross-reference multiple sources
    3. Check publication dates for currency
    4. Document source credibility in report
  </Accordion>
</AccordionGroup>
