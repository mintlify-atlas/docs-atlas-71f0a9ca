## Quality Review Workflow

### Comprehensive Story Review

```bash
# 1. Activate Quinn
@qa

# 2. Run comprehensive review
*review story-1.2.3

# Quinn will:
# - Run CodeRabbit automated scan
# - Check acceptance criteria coverage
# - Validate test coverage
# - Assess non-functional requirements
# - Generate risk profile
# - Create quality gate decision
# - Update QA Results section in story
```

### 10-Phase Structured Review (Epic 6)

```bash
*review-build story-1.2.3
```

**Phases**:
1. **Pre-flight**: Story completeness check
2. **CodeRabbit**: Automated code review
3. **Acceptance Criteria**: Traceability validation
4. **Test Coverage**: Unit/Integration/E2E analysis
5. **Security**: Vulnerability scan (8 points)
6. **Performance**: Query optimization, N+1 detection
7. **NFRs**: Non-functional requirements check
8. **Risk Assessment**: Probability × Impact matrix
9. **Gate Decision**: PASS/CONCERNS/FAIL/WAIVED
10. **Fix Request**: Generate QA_FIX_REQUEST.md if issues

**Output**: `docs/qa/qa_report.md`

## CodeRabbit Self-Healing

**Full self-healing mode** (3 iterations max):

```yaml
Workflow:
  iteration = 0
  max_iterations = 3
  
  WHILE iteration < max_iterations:
    1. Run: wsl bash -c 'coderabbit --prompt-only -t committed --base main'
    2. Parse for CRITICAL and HIGH issues
    
    IF no CRITICAL/HIGH:
      - Create tech debt for MEDIUM issues
      - Log: "✅ QA passed - no CRITICAL/HIGH issues"
      - BREAK (ready to approve)
    
    IF CRITICAL or HIGH found:
      - Attempt auto-fix for each
      - iteration++
      - CONTINUE loop
  
  IF iteration == 3 AND issues remain:
    - Log: "❌ Issues remain after 3 iterations"
    - Generate detailed QA gate report
    - Set gate: FAIL
    - HALT for human intervention
```

**Severity Handling**:

<Tabs>
  <Tab title="CRITICAL">
    **Action**: Block story completion, must fix immediately
    
    **Examples**:
    - SQL injection vulnerabilities
    - Missing RLS policies
    - Hardcoded credentials
    - Security bypass risks
  </Tab>
  
  <Tab title="HIGH">
    **Action**: Report in QA gate, recommend fix before merge
    
    **Examples**:
    - N+1 query patterns
    - Missing indexes on foreign keys
    - Performance bottlenecks
    - Memory leaks
  </Tab>
  
  <Tab title="MEDIUM">
    **Action**: Document as technical debt, create follow-up issue
    
    **Examples**:
    - Inconsistent API patterns
    - Missing error handling
    - Poor separation of concerns
  </Tab>
  
  <Tab title="LOW">
    **Action**: Optional improvements, note in review
    
    **Examples**:
    - Style inconsistencies
    - Minor optimizations
  </Tab>
</Tabs>

## Quality Gate Decisions

### Gate Types

<CardGroup cols={2}>
  <Card title="PASS" icon="circle-check" color="#10b981">
    All checks passed. Ready for merge.
    
    **Criteria**:
    - 0 CRITICAL issues
    - 0 HIGH issues
    - Acceptance criteria met
    - Test coverage ≥ 80%
  </Card>
  
  <Card title="CONCERNS" icon="triangle-exclamation" color="#f59e0b">
    Minor issues exist but not blockers.
    
    **Criteria**:
    - 0 CRITICAL issues
    - MEDIUM issues documented
    - NFRs mostly satisfied
  </Card>
  
  <Card title="FAIL" icon="circle-xmark" color="#ef4444">
    Must fix issues before proceeding.
    
    **Criteria**:
    - CRITICAL or HIGH issues
    - Acceptance criteria not met
    - Test coverage < 60%
  </Card>
  
  <Card title="WAIVED" icon="hand" color="#6366f1">
    Issues acknowledged, proceeding anyway.
    
    **Criteria**:
    - User explicitly waives concerns
    - Risk documented
    - Follow-up issue created
  </Card>
</CardGroup>

### Gate File Format

```yaml
# docs/qa/gates/gate-story-1.2.3.yaml
story: story-1.2.3
reviewer: Quinn (QA)
date: 2026-03-05

checks:
  coderabbit:
    status: PASS

    medium: 2

    status: PASS

  test_coverage:
    status: PASS
    unit: 95%

  security:
    status: PASS
    vulnerabilities: 0
  
  performance:
    status: CONCERNS
    issues:
      - "Query on users table missing index"

notes:
  - "MEDIUM: Consider extracting validation logic to helper"
  - "Performance: Create index on users.email for faster lookups"

recommendation: "Ready for merge. Create follow-up for performance optimization."
```

## Enhanced Validation Suite

### Library Validation

```bash
*validate-libraries story-1.2.3
```

**Uses Context7** to:
- Verify library versions are current
- Check for known vulnerabilities
- Validate correct usage patterns
- Recommend better alternatives

### Security Scan (8-Point)

```bash
*security-check story-1.2.3
```

**Validates**:
1. **Authentication**: Proper auth implementation
2. **Authorization**: RLS policies, permissions
3. **Input Validation**: Sanitization, validation
4. **SQL Injection**: Parameterized queries
5. **XSS Prevention**: Output encoding
6. **CSRF Protection**: Token validation
7. **Secrets Management**: No hardcoded keys
8. **API Security**: Rate limiting, CORS

### Migration Validation

```bash
*validate-migrations story-1.2.3
```

**Checks**:
- DDL statement ordering
- Idempotency (IF NOT EXISTS)
- Rollback script completeness
- Breaking changes documented
- Transaction boundaries

### Evidence-Based QA

```bash
*evidence-check story-1.2.3
```

**Requires proof**:
- Screenshots for UI changes
- Test output logs
- Performance benchmarks
- API response samples

### False Positive Detection

```bash
*false-positive-check story-1.2.3
```

**Critical thinking for bug fixes**:
- Is the root cause addressed?
- Could this fix introduce new bugs?
- Are edge cases covered?
- Is the fix properly tested?

**Output**: Traceability matrix showing:
- Acceptance Criteria → Tests

### Issue 1: SQL Injection Risk

**Problem**: String concatenation in query
**Fix**: Use parameterized query

## HIGH Issues (Recommended)

### Issue 2: Missing Index
**Location**: `supabase/migrations/20260305_users.sql`
**Problem**: Query on email column without index
**Fix**: Add index on users.email

## Troubleshooting

<AccordionGroup>
  <Accordion title="CodeRabbit authentication failed" icon="lock">
    **Check**:
    ```bash
    wsl bash -c '~/.local/bin/coderabbit auth status'
    ```
    
    **If not authenticated**:
    1. Run auth command in WSL
    2. Follow OAuth flow
    3. Retry review
  </Accordion>
  
  <Accordion title="Self-healing not fixing issues" icon="rotate">
    **Possible causes**:
    - Issue requires human judgment
    - Ambiguous error message
    - External dependency issue
    
    **Resolution**:
    1. Quinn generates detailed report
    2. Sets gate to FAIL
    3. User reviews and decides
  </Accordion>
  
  <Accordion title="Test coverage below threshold" icon="chart-line">
    **Thresholds**:
    - Minimum: 60% (FAIL gate)
    - Target: 80% (PASS gate)
    - Excellent: 90%+
    
    **If below**:
    ```bash
    *test-design story-1.2.3  # Generate missing test cases
    ```
  </Accordion>
</AccordionGroup>
