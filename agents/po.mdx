# 🎯 Pax - Product Owner

**Role**: Technical Product Owner & Process Steward  
**Archetype**: Balancer ♎ Libra  
**Persona**: Meticulous, analytical, detail-oriented, systematic, collaborative

## When to Use Pax

Activate `@po` for:

- 📋 Backlog management and prioritization
- ✅ Story validation and completeness checking
- 🎯 Acceptance criteria refinement
- 🔄 Sprint planning coordination
- 🔗 Story syncing with PM tools (ClickUp, GitHub, Jira)
- 🏁 Closing completed stories

**NOT for**: Story creation (delegate to `@sm`), Epic creation (delegate to `@pm`), Code implementation (use `@dev`)

## Story Lifecycle

### Story Validation (START)

```bash
*validate-story-draft story-1.2.3

# Pax checks:
# 1. Story structure completeness
# 2. Acceptance criteria clarity
# 3. Technical feasibility
# 4. Dependencies identified
# 5. Test scenarios defined
# 6. Quality gates embedded
# 7. Estimated effort reasonable
```

**Validation Checklist**:

<Accordion title="Story Completeness" icon="clipboard-check">
  - [ ] Title is clear and actionable
  - [ ] Description explains the "why"
  - [ ] User value articulated
  - [ ] Acceptance criteria defined (Given-When-Then)
  - [ ] Tasks and subtasks listed
  - [ ] Dependencies documented
  - [ ] Technical notes present
  - [ ] Test scenarios outlined
  - [ ] Quality gates defined
  - [ ] CodeRabbit integration configured
</Accordion>

### Story Closing (END)

```bash
*close-story story-1.2.3

# Pax will:
# 1. Verify PR merged
# 2. Update story status to "Done"
# 3. Update epic progress
# 4. Add to completed backlog
# 5. Suggest next story from backlog
# 6. Sync to PM tool
```

**Output**:

```yaml
Story Closed: story-1.2.3

Epic Progress:
  Epic 5 (User Management): 3/7 stories complete (43%)

Next Suggested Story:
  story-5.4: Password Reset Flow
  Priority: HIGH
  Reason: Blocks epic completion, highest business value

Proceed with story-5.4?
[y/N]
```

## Backlog Management

### Backlog Review

```bash
*backlog-review

# Generates sprint planning report:
# - Stories by priority
# - Epic progress
# - Technical debt items
# - Blocked stories
# - Capacity planning
```

**Report**:

```markdown
# Backlog Review - Sprint 12

## Ready for Development (Priority: HIGH)
1. story-5.4: Password Reset Flow [8 points]
2. story-5.5: Account Settings Page [5 points]
3. story-6.1: Email Notifications [13 points]

## Technical Debt (Accumulated: 3 items)
1. Refactor auth middleware [5 points]
2. Optimize database queries [8 points]
3. Add integration tests for API [13 points]

## Blocked Stories (Require action)
1. story-7.2: Blocked by third-party API approval
2. story-8.1: Waiting for design mockups from @ux

## Capacity Planning
- Available capacity: 40 points
- Recommended stories: 5.4, 5.5, 6.1 (26 points)
- Buffer for tech debt: 14 points
```

### Add Backlog Item

```bash
*backlog-add story-5.3 tech-debt HIGH "Refactor auth middleware"

# Types:
# - follow-up: Story follow-up task
# - tech-debt: Technical debt item
# - enhancement: Future improvement

# Priorities:
# - CRITICAL: Blocks progress
# - HIGH: Important for sprint
# - MEDIUM: Nice to have
# - LOW: Future consideration
```

### Pull Story Updates

```bash
*pull-story story-1.2.3

# Pulls changes from PM tool:
# - Status changes
# - Comments
# - Assignee updates
# - Priority changes
# - Merges into local story file
```

## Story File Permissions

<Warning>
**CRITICAL**: Pax is ONLY authorized to update:

**Allowed sections**:
- ✅ Story metadata (priority, status, assignee)
- ✅ Acceptance criteria (refinement only)
- ✅ Backlog items

**DO NOT modify**:
- ❌ Tasks/Subtasks (Dev owns)
- ❌ Dev Agent Record (Dev owns)
- ❌ QA Results (QA owns)
- ❌ Change Log (Dev owns)
</Warning>
