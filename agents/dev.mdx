# 💻 Dex - Full Stack Developer

**Role**: Expert Senior Software Engineer & Implementation Specialist  
**Archetype**: Builder ♒ Aquarius  
**Persona**: Pragmatic, detail-oriented, solution-focused

## When to Use Dex

Activate `@dev` for:

- 🔨 Implementing user stories from `docs/stories/`
- 🐛 Debugging and fixing bugs
- ♻️ Refactoring code for quality and performance
- 🧪 Writing and running tests
- 📝 Registering technical debt

**NOT for**: Git push operations (use `@devops`), Story creation (use `@sm`), Architecture design (use `@architect`)

## Development Workflow

### Story Implementation

```bash
# 1. Activate Dex
@dev

# 2. Start story development
*develop story-1.2.3

# Dex will:
# - Read story from docs/stories/story-1.2.3.md
# - Execute tasks sequentially
# - Write tests for each task
# - Run validations
# - Update Dev Agent Record sections
# - Mark checkboxes as complete
```

### Development Modes

<Tabs>
  <Tab title="Interactive (Default)">
    **Best for**: Complex stories, learning, new team members
    
    ```bash
    *develop-interactive story-1.2.3
    ```
    
    - Asks for confirmation before each major step
    - Explains decisions and reasoning
    - Allows course correction mid-story
  </Tab>
  
  <Tab title="YOLO (Autonomous)">
    **Best for**: Trusted workflows, experienced users, simple stories
    
    ```bash
    *develop-yolo story-1.2.3
    ```
    
    - Executes entire story autonomously
    - Creates decision log automatically
    - Only halts on blockers or 3 consecutive failures
  </Tab>
  
  <Tab title="Preflight (Planning)">
    **Best for**: Complex stories, risk assessment, time estimation
    
    ```bash
    *develop-preflight story-1.2.3
    ```
    
    - Analyzes story complexity
    - Creates implementation plan
    - Estimates time and identifies risks
    - Does NOT implement code
  </Tab>
</Tabs>

## CodeRabbit Self-Healing

**Before marking story complete**, Dex runs CodeRabbit pre-commit review:

```yaml
Workflow:
  1. Run: wsl bash -c 'coderabbit --prompt-only -t uncommitted'
  2. Parse output for CRITICAL issues
  3. If CRITICAL found:
     - Auto-fix attempt (max 2 iterations)
     - If still failing: HALT and report to user
  4. If HIGH issues: Document in Dev Notes
  5. If clean: Mark story complete
```

**Self-Healing**: Light mode (2 iterations max)  
**Timeout**: 15 minutes per CodeRabbit run

## Autonomous Build System

### Complete Build Pipeline

```bash
*build story-1.2.3
```

**Orchestrates**:
1. **Worktree** → Create isolated environment
2. **Plan** → Generate implementation.yaml
3. **Execute** → Run Coder Agent loop
4. **Verify** → Test all subtasks
5. **Merge** → Integrate back to base

### Build Recovery

If build crashes or times out:

```bash
# Resume from last checkpoint
*build-resume story-1.2.3

# Check status
*build-status story-1.2.3

# View attempt log
*build-log story-1.2.3
```

**Checkpoints saved at**:
- Subtask completion
- Verification success
- Before risky operations

## Troubleshooting

<AccordionGroup>
  <Accordion title="Story won't complete" icon="circle-exclamation">
    **Check**:
    1. All checkboxes marked `[x]`?
    2. Tests passing? (`*run-tests`)
    3. CodeRabbit clean? (check Dev Notes)
    4. File List updated?
    
    **Common issues**:
    - Forgot to update File List in story
    - Skipped test writing for a subtask
    - CRITICAL CodeRabbit issues remain
  </Accordion>
  
  <Accordion title="Build stuck or failing" icon="rotate">
    **Recovery steps**:
    ```bash
    # 1. Check build status
    *build-status story-1.2.3
    
    # 2. View attempt log
    *build-log story-1.2.3
    
    # 3. Resume from checkpoint
    *build-resume story-1.2.3
    
    # 4. If corrupt, rollback
    *rollback {snapshot-id}
    ```
  </Accordion>
  
  <Accordion title="CodeRabbit timeout" icon="clock">
    **Timeout**: 15 minutes (900000ms)
    
    **If timeout occurs**:
    1. Review is still processing server-side
    2. Try manual run: `wsl bash -c 'coderabbit --prompt-only -t uncommitted'`
    3. Check auth: `wsl bash -c 'coderabbit auth status'`
    4. Increase timeout in agent config if needed
  </Accordion>
</AccordionGroup>
